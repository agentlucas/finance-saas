## Lessons Learned
1. Railway deployment works well with monorepos when root `package.json` has proper build/start scripts
2. nixpacks.toml can help but isn't always necessary if package.json is configured correctly
3. Vercel monorepo deployments may need special configuration or deploying from subdirectory
4. Database migrations must be run from Railway environment, not locally
5. TypeScript strictNullChecks requires explicit null checks even when logically impossible
